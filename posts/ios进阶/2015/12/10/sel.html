Method结构, SEL, IMP理解

一直在代码中使用@Selector(method:)，但其原理也是有必要去了解的。

定义

下面是method相关的定义

 1 typedef struct objc_method \*Method;
 2 //编译器依据类中定义的方法为该类产生一个或更多这种这种结构.
 3 //一个类的实现可以分散在一个文件中不同部分,同时类别可以分散在不同的模块中.
 4 //为了处理这个问题,使用一个单独的方法链表 
 5 struct objc_method {
 6     SEL method_name;
 7     char \*method_types;
 8     IMP method_imp;
 9 } 
10 //
11 struct objc_method_list {
12       struct objc_method_list \*obsolete         OBJC2_UNAVAILABLE;
13       int method_count                          OBJC2_UNAVAILABLE;
14   #ifdef \_\_LP64__
15       int space                                 OBJC2_UNAVAILABLE;
16   #endif
17       /* variable length structure \*/
18       struct objc_method method_list[1]         OBJC2_UNAVAILABLE;
19  } 
20  //
21 struct objc_method_description {
22       SEL name;               /* < The name of the method \*/
23       char \*types;            /* < The types of the method arguments */
24 
25 };
26 //
27 struct objc_method_description_list {
28       int count;
29       struct objc_method_description list[1];
30 };

根据结构体objc_method可以看出，每个方法有3个属性

  • SEL：selector 的简写,俗称方法选择器,实质存储的是方法的名称，方法名为此方法的签名，有着相同函数名和参数名的方法有着相同的方法名。
  • char method_types：描述方法的参数列表. 在运行时注册选择器时使用.那时候方法名就会包含方法的参数列表。
  • IMP：implement 的简写,俗称方法实现,看源码得知它就是一个函数指针，为方法具体实现代码块的地址，可像普通C函数调用一样使用IMP

IMP 和 SEL 的结构如下：

IMP

1 id (*IMP)(id, SEL, ...)

SEL

1 typedef struct objc_selector *SEL;
2 struct objc_selector
3 {
4   void *sel_id;
5   const char *sel_types;
6 };

发送消息

当一个receiver接收消息时，会触发 objc_msgSend(receiver, SEL op, ...); ，首先去runtime底层代码去查看，这些参数的来源，每一个receiver都是objc_object结构体的实例，

1 struct objc_objc {
2 	Class isa  OBJC_ISA_AVAILABILITY;
3 }

通过 isa 指针找到当前对象的 class，并在class的 method list中寻找发送的消息, 结构体objc_method_list，根据官方文档的说法，这是一个包含了多个 objc_method 的数组，`objc_method 这个结构体中包含了2个重要信息—SEL 和 IMP, receiver接收消息时传的参数是SEL类型，SEL主要用来标注方法的名称，IMP则是方法指针，因为每个方法都有自己的地址，这种方式直接找到方法的地址，效率最高。 这样就完成了消息的发送过程

简单来说，当你发送消息时，首先回去找它对应的方法list，找到后就会使用相应的方法。
大致是这样的流程，当然细节也很多

  1. 它首先找到 SEL 对应的方法实现 IMP。因为不同的类对同一方法可能会有不同的实现，所以找到的方法实现依赖于消息接收者的类型。

  2. 然后将消息接收者对象(指向消息接收者对象的指针)以及方法中指定的参数传递给方法实现 IMP。

  3. 最后，将方法实现的返回值作为该函数的返回值返回。
