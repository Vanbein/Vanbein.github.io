Objective-C的继承和多态

面向对象编程之所以成为主流的编程思想和他的继承和多态是分不开的，只要是面向对象语言都支持继承和多态，当然不同的OOP语言之间都有其特点。

OC中和Java类似，不支持多重继承，但OOP语言C++就支持多继承，为什么OC不支持多继承稍后将会提到

在Objective-C中 super 是指向直接父类的指针，而self是指向本身的指针，self就相当于java中的this指针。在OC中写类时可以在 @implementation 中定义哪些在 @interface 中无相应声明的方法，但这个方法是私有的，仅在类的实现中使用。

一、NSObject

在Objectiv-C中几乎所有的类都是继承自NSObject类，NSObject类中存在大量功能强大的方法。下面对NSObject类中的各种方法进行试验和介绍：

1. +(void) load;

  • 类加载到运行环境时调用该方法

测试：在子类中重写load方法来进行测试, 当重写完load方法，在mian方法中不需要任何实例化任何对象

当类被加载时load就会别调用，load是类方法，可以直接被类调用

1 //重写NSObject中的load方法
2 +(void) load
3 {
4     NSLog(@"load方法被调用了！");
5 }
6 //运行结果： load方法被调用了！

2. +(void) initialize;

  • 在类第一次使用该类时调用该方法，第二次就不调用了

测试：重写initalize方法，

1 //重写initialize方法，会在类第一次使用时调用
2 +(void) initialize
3 {
4     NSLog(@"initialize方法被调用了！");
5 }
6 //运行结果
7 //load方法被调用了！
8 //initialize方法被调用了！

3. +(id) alloc: 、 -(id) init: 和 new

  • +(id) alloc: 返回一个已经分配好的内存对象；
  • -(id) init: 对已经分配了内存的实例进行初始化；
  • new 同时调用了alloc 和 init

可以在子类中把alloc进行重写，然后观察运行情况

举个例子

 1 //重写alloc方法
 2 +(id) alloc
 3 {
 4    NSLog(@"alloc函数被调用啦");
 5     return [super alloc];
 6 }
 7 //重写init
 8 -(id) init
 9 {
10     NSLog(@"init被调用了");
11     self = [super init];
12     return self;
13 }
14 //实例化类
15 ObjectTest *o1 = [[ObjectTest alloc] init];
16 // 
17 ObjectTest *o2 = [ObjectTest new];
18 //运行结果：
19 //alloc函数被调用啦
20 //init被调用了
21 //alloc函数被调用啦
22 //init被调用了

4. -(void)dealloc

  • 释放对象自身所占有的内存；

5. -(Class)class 和 -(Class)superclass

  • -(Class)class或者 +(Class)class 返回当前对象的所属类;
  • -(Class)superclass 或者 +(Class)superclass返回当前类的父类
1 //返回当前对象所对应的类
2 NSString *className =(NSString *) [self class];
3 NSLog(@"%@类的display方法", className);
4 // 
5 //返回当前对象所对应的父类
6 NSString *superClassName = (NSString *) [self superclass];
7 NSLog(@"%@类的父类是%@", className, superClassName);

6. -(BOOL)isKindOfClass : (Class)aClass

  • 判断某个实例是否属于某个类或者子类的对象

示例代码如下：

1 //isKindOfClass的用法
2 BOOL b =  [objct isKindOfClass:[ObjectTest class]];
3 if (b == YES) {
4     NSLog(@"objct是ObjectTest类的对象");
5 }

7. -(BOOL)isMemberOfClass:(Class)aClass;

  • 这只能判断摸个实例是否属于某个类，不能判断是否属于某个父类；
1 //isMemberOfClass
2 BOOL result = [o2 isMemberOfClass:[NSObject class]];
3 if (result == NO) {
4     NSLog(@"isMemberOfClass不能判断是否为NSObject子类的对象");
5 }

8. -(NSString *) description;

  • 返回字符串形式对象的描述，方便调试
1 //description
2 NSString *descript = [o2 description];
3 NSLog(@"%@", descript);
4 //输出结果： <ObjectTest: 0×100206680>

9. -(NSUInteger) hash;

  • 返回对象的哈希值；
1 //hash的用法
2 NSUInteger hash = [o2 hash];
3 NSLog(@"%p", hash);
4 //输出结果：2014-07-30 11:40:35.685 HelloOC[1130:303] 0x100107b70

10. -(BOOL) isEqual:(id)object;

  • 比较两个对象是否相同，默认是使用地址进行比较的，且hash值一定要相同
 1 //isEqual的用法
 2 NSString *str1 = @"111";
 3 NSString *str2 = str1;
 4 if ([str2 isEqual:str1] == YES)
 5 {
 6     NSLog(@"str2 == str1");
 7 }
 8 else
 9 {
10     NSLog(@"str2 != str1");
11 }

二、Objective-C中的继承

继承是is-a的关系，比如猫是一个动物，那么动物是父类，而猫是动物的子类。子类具有父类的属性和 行为，以及自身的属性和行为，也就是说“父类更一般，子类更具体”。用一个富二代的类来说明一下类的继承。

1.先定义一个富人的类

代码说明：
  1. 声明访问权限为@protected的三个属性，分别为三个属性用@property 加上 getter 和setter 方法
  2. 并为该类创建便利初始化方法和便利构造器
  3. 为富人类定义一个刷卡方法
 1 @interface Richer : NSObject
 2 {
 3     @protected
 4     NSString *name;
 5     int age;
 6     NSString *gender;
 7 }
 8 // 
 9 //定义富人的姓名，年龄，性别，并为其提供getter和setter方法
10 @property (copy, nonatomic) NSString *name;
11 @property (assign, nonatomic) int age;
12 @property (copy, nonatomic) NSString *gender;
13 // 
14 //定义便利初始化方法
15 -(id) initWithName : (NSString *)vName
16             AndAge : (int)vAge
17          AndGender : (NSString *)vGender;
18 // 
19 //定义便利构造器
20 +(id) richerWithName : (NSString *)vName
21               AndAge : (int)vAge
22            AndGender : (NSString *)vGender;
23 // 
24 //定义刷卡方法
25 -(void) poss;
26 //
27 @end

2.为富人类编写实现代码

代码说明：
  1. 用@synthesize实现getter和setter方法
  2. 实现便利初始化方法，用[ super init ]初始化富人类的直接父类，也就是NSObject
  3. 使用便利构造器返回实例化并初始化后的对象
 1 #import "Richer.h"
 2 @implementation Richer
 3 //实现getter和setter方法
 4 @synthesize name, age, gender;
 5 //实现便利初始化函数
 6 -(id) initWithName : (NSString *)vName
 7             AndAge : (int)vAge
 8          AndGender : (NSString *)vGender
 9 {
10     if (self = [super init])
11     {
12         self->name = vName;
13         self->age = vAge;
14         self->gender = vGender;
15     }
16     return self;
17 }
18 //实现便利构造器
19 +(id) richerWithName:(NSString *)vName
20               AndAge:(int)vAge
21            AndGender:(NSString *)vGender
22 {
23     Richer *richer = [[Richer alloc] initWithName:vName
24                                            AndAge:vAge
25                                         AndGender:vGender];
26     return richer;
27 }
28 //实现刷卡方法
29 -(void) poss
30 {
31     NSLog(@"%@有钱你就刷吧", name);
32 }@end

4.各种方法的实现

代码说明:
  1. 在编写便利初始化方法时利用super来调用父类的便利初始化方法来把继承到的父类的方法进行初始化
  2. 用self给新添加的属性进行初始化
 1 #import "Richer2nd.h"
 2 // 
 3 @implementation Richer2nd
 4 //实现属性的getter和setter方法
 5 @synthesize hobby;
 6 // 
 7 //编写便利初始化函数，复用父类的便利初始化方法
 8 -(id)initWithName:(NSString *)vName
 9            AndAge:(int)vAge
10         AndGender:(NSString *)vGender
11          AndHobby:(NSString *)vHobby
12 {
13     if (self = [super initWithName:vName AndAge:vAge AndGender:vGender]) {
14         self->hobby = vHobby;
15     }
16     return self;
17 }
18 // 
19 //编写便利构造函数
20 +(id)richer2ndWithName:(NSString *)vName
21                 AndAge:(int)vAge

​## 三、Objective-C中的多态

多态简单的说就是对于不同对象响应同一个方法时做出的不同反应。在 OC中动态类型id是实现多态的一种方式，id是一个独特的数据类型，可以转换为任何数据类型，上面的富人和富二代可以这样定义

 1 id richer = nil;
 2 // 
 3 //测试富人类
 4 richer = [Richer richerWithName:@"Bill" AndAge:40 AndGender:@"Man"];
 5 [richer poss];
 6 // 
 7 //测试富二代的类
 8 richer = [Richer2nd richer2ndWithName:@"BILL`s son" AndAge:16 AndGender:@"男" AndHobby:@"飙车"];
 9 [richer poss];
10 [richer myHobby];

上面程序的输出结果和继承部分的结果是一致的；

​多态的另一个例子: Animal是父类，子类有Cat 和 Dog,子分别重写了父类中的eat方法；实例化对象的时候可以用下面的方法：

1 Animal *animal = nil;
2  //实例化猫的对象
3 animal = [Cat new];
4 [animal eat];
5  //实例化狗的对象
6 animal = [Dog new];
7 [animal eat];
面向对象编程中的OCP原则和LSP原则
  • OCP : Open Closed Principle原则， 对扩展开放，对修改关闭
  • LSP ：里氏代换原则，任何基类可以出现的地方，子类一定可以出现。
